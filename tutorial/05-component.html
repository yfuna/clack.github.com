Sorry. I told a lie in Chapter 3. --Clack application is just a function--. The truth is, it's not always a function. See this example:

;; importing symbols for readability.
(import '(clack:call clack:<component>))

(defclass <sample-app> (<component>) ())

(defmethod call ((this <sample-app>) env)
  (declare (ignore env))
  '(200
    (:content-type "text/plain")
    ("Hello, Clack!")))

(clack:clackup
  (make-instance '<sample-app>))

This is much similar to what we saw before, isn't it? The content of the function has moved into the method call, and an instance of <sample-app> is now being pass to clackup.

In a nutshell, clackup takes a function or an instance of a subclass of <component> as a Clack application implemetation. All components must have call implemented, which is called for each request.

You may not understand how we handle these stuffs yet. I will tell it using more practical examples in succeeding chapters. Please just remember clackup can take a CLOS instance for now.
