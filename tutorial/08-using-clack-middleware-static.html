Chapter 8: Using Clack.Middleware.Static

English | 日本語

Clack contains these useful middlewares.

  • Clack.Middleware.Static - Returns static files.
  • Clack.Middleware.Logger - Logs behaviours of Clack application or middlewares.
  • Clack.Middleware.Session - Manages sessions.
  • Clack.Middleware.Conditional - Provides conditional wrapper.
  • Clack.Middleware.Auth.Basic - Provides basic authentication.

Clack.Middleware.Csrf which protects from CSRF attacks is also available as a contrib package.

In this chapter, I would like to introduce Clack.Middleware.Static as a real example of a Clack middleware. It returns static files if the given request matches with the specified pattern.

Here's an example source code to show how to use it. This wrapped application returns "Hello, Clack!" for most of requests, but only when the request path matches /public/*, a file in /path/to/static-files/* will be served.

(import '(clack:wrap
          clack.middleware.static:<clack-middleware-static>))

(defun app (env)
  (declare (ignore env))
  '(200
    (:content-type "text/plain")
    ("Hello, Clack!")))

(defvar mw
    (make-instance '<clack-middleware-static>
       :path "/public/"
       :root #p"/path/to/static-files/"))

(clack:clackup
  (wrap mw #'app))

Middlewares implement a method wrap which is for generating a wrapped application. An application can also be wrapped multiple Middlewares. For that, call wrap multiple times, like this.

(clack:clackup
  (wrap mw3
    (wrap mw2
      (wrap mw1 #'app))))

However, these nested wraps are tedious. Clack has the solution to write it with a DSL style syntax. That's the topic of the next chapter.
